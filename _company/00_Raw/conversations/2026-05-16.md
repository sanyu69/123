## 🚀 최적 기술 스택 제안 (The Optimal Stack)

웹 기반 캐주얼 액션 게임에 가장 적합한 스택은 **'최소한의 오버헤드로 최대의 시각 효과를 내는 것'**입니다. 따라서 프레임워크 의존도를 최소화하고, 순수 브라우저 API 활용을 중심으로 제안합니다.

### 🥇 1순위 추천: Vanilla JavaScript + HTML Canvas (The Control Stack)

이 스택은 **성능, 통제력, 배포 용이성** 측면에서 가장 우수합니다.
*   **장점:**
    *   **최고의 성능 제어:** 게임 루프와 렌더링을 직접 제어하므로 불필요한 오버헤드가 없습니다. 이는 네온 효과나 복잡한 입자 시스템 구현에 절대적으로 유리합니다.
    *   **경량성:** 외부 라이브러리 의존성이 없어 빌드 과정이 간소화되며, 최종 파일 크기가 작습니다 (PWA 환경에도 적합).
    *   **직접적인 이해:** 모든 픽셀과 속도가 어디서 오는지 명확히 알 수 있어 디버깅이 매우 용이합니다.
*   **단점:** 구조를 잡기 위한 초기 설정(게임 루프, 충돌 감지 로직)을 직접 구현해야 합니다.

### 🥈 2순위 대안: Phaser.js (The Structured Stack)

프로토타입의 속도를 극대화하고 싶다면, 검증된 2D 게임 프레임워크인 Phaser를 고려할 수 있습니다.
*   **장점:**
    *   **개발 속도:** 물리 엔진, 스프라이트 관리, 씬(Scene) 관리가 내장되어 있어 초기 개발 속도가 매우 빠릅니다.
    *   **커뮤니티 지원:** 문제 발생 시 참고할 자료가 풍부합니다.
*   **단점:** Vanilla JS보다 상대적으로 오버헤드가 있으며, 네온 특유의 독특한 렌더링 효과를 커스터마이징할 때 추가적인 노력이 필요할 수 있습니다.

프로토타입은 **단일 HTML 파일**로 구성하여 즉시 실행 가능하고 배포 용이성을 확보해야 합니다. 이는 PWA(Progressive Web App) 환경에 가장 적합합니다.

### ⚙️ 핵심 파일별 역할 분담

1.  **`index.html` (The Container & UI)**
    *   게임 캔버스(`canvas` 태그)를 정의합니다.
    *   외부 리소스(CSS, JS 파일)를 로드합니다.
    *   게임 시작/종료 화면 및 네온 UI 요소를 담는 HTML 구조를 담당합니다.

2.  **`style.css` (The Vibe Setter)**
    *   전체 배경에 적용될 **어두운 배경색(Dark Mode)**과 **네온 색상 정의**를 설정합니다.
    *   캔버스 및 UI 요소에 대한 기본적인 시각적 스타일을 정의하여 아트 디렉션(`Designer`의 작업)을 구현합니다.

3.  **`script.js` (The Engine & Logic)**
    *   **Game Loop:** `requestAnimationFrame`을 사용하여 부드러운 게임 루프를 관리합니다.
    *   **Input Handler:** 마우스/키보드 입력을 받아 캐릭터의 방향이나 파워업 선택을 처리합니다.
    *   **Entity Management:** 적(Enemies), 플레이어, 아이템 등의 상태와 위치를 관리하는 클래스 또는 객체를 정의합니다.
    *   **Rendering Logic:** `Canvas API`를 사용하여 모든 엔티티를 캔버스에 네온 스타일로 그립니다.
    *   **Game State Management:** 점수, 생명력, 현재 활성화된 무기 등의 상태 변화를 추적하고 업데이트합니다.

다음은 게임 플레이에 필수적인 세 가지 핵심 화면의 레이아웃 구조입니다. 모든 요소는 네온 효과와 깊이감을 강조해야 합니다.

#### A. 메인 메뉴 (Main Menu)
**목표:** 플레이어에게 다음 행동을 명확하게 제시하고, 브랜드 분위기를 즉각적으로 전달한다.
*   **레이아웃:** 중앙 집중형 (Centered Focus). 배경은 깊은 블랙 또는 아주 어두운 네이비 톤.
*   **핵심 요소:**
    1.  **로고/타이틀:** 화면 상단 중앙에 가장 크고 강렬한 네온 효과 적용. (`Neon_Vibe_Palette`의 메인 컬러 사용)
    2.  **게임 시작 버튼 (Start Game):** 가장 시각적으로 강조되는 CTA(Call to Action). 깜빡이는 듯한 애니메이션 추가 권장.
    3.  **설정 (Settings):** 보조적인 네온 라인으로 처리하여 미니멀하게 배치.
    4.  **종료/옵션:** 오른쪽 하단에 작게 배치.

#### B. 인게임 HUD 및 레벨업 UI (In-Game HUD & Level Up UI)
**목표:** 실시간 정보 전달과 성취감을 극대화한다. 플레이어가 게임의 진행 상황을 명확하게 인지하도록 한다.
*   **레이아웃:** 화면 가장자리(Edge)를 활용하여 정보를 배치하고, 중앙은 액션에 집중시킨다.
*   **핵심 요소:**
    1.  **체력/자원 바 (Health/Resource Bar):** 화면 좌측 상단 또는 하단에 고정. 네온 라인으로 구성되며, 위험 시 색상 변화(예: 빨강에서 주황)를 통해 긴장감을 부여.
    2.  **경험치/레벨 표시:** 화면 중앙 상단에 배치. 레벨업 시 폭발적인 네온 효과와 함께 팝업 메시지 발생.
    3.  **스코어보드 (Scoreboard):** 현재 점수 및 보너스 정보는 미세하게 움직이는 듯한 글리치(Glitch) 효과를 적용하여 '네온 리플레이' 컨셉을 강화.

#### C. 설정 화면 (Settings Screen)
**목표:** 복잡한 설정을 시각적으로 깔끔하고 테마에 맞게 정리한다.
*   **레이아웃:** 카드 기반 레이아웃 (Card-based Layout). 각 항목은 독립적인 네온 박스 형태로 구성.
*   **핵심 요소:**
    1.  **배경:** 어두운 배경 위에 반투명한 네온 오버레이를 사용하여 깊이감을 부여.
    2.  **항목 목록:** 폰트의 경계를 활용하여 항목을 구분하고, 선택 시 해당 항목에만 강렬한 색상(예: 시안)이 발현되도록 디자인한다.

        function updateHUD() {
            // HUD 업데이트 로직 자리 표시자
            document.querySelector('.stat-bar:nth-child(1)').textContent = `Score: ${score}`;
            document.querySelector('.stat-bar:nth-child(2)').textContent = `Health: ${health}%`;

        function gameLoop() {
            if (gameState !== 'PLAYING') return;

            // 1. 상태 업데이트 (Logic)
            updateEntities(); // 엔티티 위치, 충돌 계산 등
            handleInput();     // 사용자 입력 처리
            updateHUD();       // HUD 데이터 업데이트

            // 2. 그리기 (Render)
            ctx.clearRect(0, 0, canvas.width, canvas.height);
            drawEntities(); // 모든 객체 그리기
            drawHUD();       // 네온 스타일의 UI 요소 그리기

            requestAnimationFrame(gameLoop);
        }

        function drawEntities() {
            // 엔티티 시각화 자리 표시자 (네온 효과를 적용하여 구현)
            entities.forEach(entity => {
                ctx.fillStyle = entity.color || 'cyan';
                ctx.shadowColor = entity.color;
                ctx.shadowBlur = 10;
                ctx.beginPath();
                ctx.arc(entity.x, entity.y, entity.radius, 0, Math.PI * 2);
                ctx.fill();
            });
        }

2.  **IAP 중심:** 광고는 수익화 수단이 아닌 **사용자 경험 저해 요소**로 간주하고, IAP을 통해 사용자가 원치 않는 요소를 제거할 수 있는 '프리미엄 경험'으로 포지셔닝합니다.

## 2. 초기 KPI 목표 설정 (Key Performance Indicators)

수익화 모델이 IAP 중심이므로, **사용자 확보(Acquisition)**보다 **사용자 유지 및 전환(Retention & Conversion)**에 초점을 맞춥니다.

| KPI 항목 | 측정 목표 (3개월 내) | 전략적 근거 |
| :--- | :--- | :--- |
| **DAU/MAU 비율** | 30% 이상 달성 | 높은 일일 활성 사용자 비율은 게임 루프의 중독성을 증명하며, IAP 구매 기회를 극대화합니다. (Engagement 중심) |
| **IAP 전환율 (CR)** | 초기 7일 내 2% 이상 | MVP 단계이므로, 핵심 기능(광고 제거 또는 미적 요소)에 대한 가치를 명확히 인지시킨 후 구매를 유도하는 테스트 지표입니다. |
| **LTV (Lifetime Value)** | $15 이상 달성 | VIP 티어의 성공적인 도입을 통해 장기적인 고객 생애 가치를 측정하며, 이는 광고 기반 모델보다 훨씬 높은 수익성을 제공합니다. |
| **ARPU (Average Revenue Per User)** | $3.50 이상 | 초기 설정된 가격 정책과 Pro/VIP 전환율을 고려했을 때 목표하는 평균 사용자당 수익입니다. |

### 🎯 수익화 전략 초안: 경험 중심의 Funnel 설계

1.  **Awareness (인지):** 네온 리플레이 컨셉의 강력한 비주얼(Designer)과 매력적인 세계관(Writer)을 활용하여 소셜 미디어에서 바이럴을 유도합니다. (광고 지양)
2.  **Acquisition (유입):** 무료 플레이를 통해 게임 루프에 즉시 몰입하게 한 후, **'불편함 해소'**를 통한 IAP 유도를 핵심 동력으로 삼습니다.
3.  **Conversion (전환):**
    *   **Pain Point:** 사용자는 '광고'나 '지속적인 아트 선택의 피로감'을 불편함으로 느낍니다.
    *   **Solution:** IAP은 이러한 **불편함을 제거하고, 원하는 비주얼과 경험을 즉시 제공하는 지름길**으로 포지셔닝합니다.
4.  **Retention (유지):** VIP 티어의 독점 콘텐츠와 지속적인 보상 시스템을 통해 커뮤니티 및 충성도를 구축하여 장기적인 LTV를 확보합니다.

최종 목표는 도시 전체에 흩어진 모든 파편화된 기억을 통합하여, 자신만의 완벽하고 영원한 '네온 자아(Neon Self)'를 완성하는 것입니다. 이 과정에서 플레이어는 단순한 생존자를 넘어, 네온 시티의 역사를 기록하는 존재가 됩니다.
