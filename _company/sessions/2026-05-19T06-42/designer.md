# 📄 MVP 아트 에셋 최종 디자인 명세 (Basic/Pro)

## 1. 디자인 시스템 확정 (Foundation)

모든 아트 에셋은 다음의 핵심 디자인 시스템을 공유해야 합니다.

*   **컬러 팔레트 (Neon_Vibe_Palette):**
    *   **Primary Neon (기본 네온):** `#00FFFF` (Cyan/Aqua) - 배경 및 주요 강조선
    *   **Secondary Glow (보조 광택):** `#FF00FF` (Magenta) - 인앱 구매 버튼, 활성 상태 표시
    *   **Background Base (배경 베이스):** `#1A1A2E` (Deep Indigo/Dark Blue) - 전체 UI 배경
    *   **Text Primary (주요 텍스트):** `#FFFFFF` (Pure White)
    *   **Accent Glow (보조 강조):** `#00FF7F` (Spring Green) - 레벨업, 성공 피드백

*   **타이포그래피:**
    *   **Font Family:** `Orbitron` (Headlines/Titles), `Roboto Mono` (Body Text/Data)
    *   **Usage:** 헤더 및 핵심 정보는 `Orbitron`을 사용하여 네온 느낌 강조. 데이터 표시는 `Roboto Mono`를 사용하여 시스템적인 느낌 부여.

## 2. 아트 에셋 차별화 명세 (Minimal Differentiation)

시각적 차별화를 최소화하고 기능적 차이점에만 집중하여 제작합니다.

### 2.1. Basic 티어 에셋 명세
**목표:** 기본 게임 플레이 및 IAP 흐름에 필요한 핵심 시각 요소 제공.

*   **UI 스타일:** 'Neon_Vibe_Palette' 전체를 적용하며, 가장 기본적인 네온 효과만 사용.
*   **아이콘/프레임:** 모든 UI 프레임은 단순한 선(Outline) 기반으로 제작하여 깔끔함을 유지합니다. (Pro 티어에서만 채움(Solid) 또는 추가적인 복잡한 광원을 적용)
*   **핵심 아트 요소 (예시):**
    *   게임 시작 화면 배경: `#1A1A2E` 베이스에 `#00FFFF` 테두리 강조.
    *   레벨업 텍스트: `Orbitron`, 색상 `#00FF7F`.

### 2.2. Pro 티어 에셋 명세
**목표:** Basic 대비 시각적 만족도를 높이고 프리미엄 경험을 제공. (차별화는 '깊이'와 '강도'에 집중)

*   **UI 스타일:** Basic과 동일한 레이아웃 및 색상 팔레트를 유지하되, **광원(Glow)의 강도와 깊이를 증가**시킵니다.
*   **아이콘/프레임:** 주요 UI 요소(버튼, 프레임)는 채워진(Solid) 네온 효과를 적용하여 시각적 무게감을 더합니다.
    *   **Primary Neon (기본):** `#00FFFF` (선 강조 유지)
    *   **Secondary Glow (강화):** `#FF00FF` (버튼 및 핵심 프레임에 사용, 강도 최대화)
*   **핵심 아트 요소 (예시):**
    *   게임 시작 화면 배경: 깊은 네온 그림자 효과(`Box-shadow` 극대화).
    *   레벨업 텍스트: `Orbitron`, 색상 `#FF00FF`로 강조.

## 3. 개발 전달 지침 (For Koda-ri)

코다리에게는 다음의 지침을 전달하여 아트 에셋 제작을 시작하도록 요청합니다.

1.  **시스템 준수:** 모든 아트 에셋은 `Neon_Vibe_Palette` 규칙(특히 색상 코드와 폰트 사용)을 절대적으로 준수해야 합니다.
2.  **차별화 최소화:** Basic과 Pro 간의 차이는 **'광원의 강도 및 채움 여부'**로만 한정하며, 레이아웃이나 기본 형태는 동일하게 유지하여 개발 속도를 최우선으로 확보하십시오.
3.  **최종 출력:** 최종 산출물은 Figma 또는 이미지 포맷으로 제공되어야 하며, 각 에셋에 사용된 색상 코드와 폰트 명세를 별도로 첨부해야 합니다.
