Phase 2 핵심 기능 프로토타입 구현 계획을 시스템 안정성($R_{Stability}$) 확보에 초점을 맞춰 업데이트하고, 기술적 마일스톤과 예상 소요 시간을 산출하여 보고드리겠습니다.

이 계획은 이전에 논의된 데이터 파이프라인 및 API 안정성 검증 로직 구현을 기반으로 하며, $R_{Stability}$를 최우선 전제 조건으로 설정합니다.

### Phase 2: 시스템 안정성 확보 로드맵 (안정성 중심)

현재 목표는 웹사이트 이벤트 로그와 IAP 트랜잭션 데이터의 **수집, 안정적 저장, 그리고 실시간 검증** 체계를 구축하는 것입니다.

#### 1단계: 데이터 수집 및 API 엔드포인트 구현 (Foundation)
이 단계는 이전 세션에서 시작된 초기 엔드포인트 구현을 완성하고 데이터 흐름을 확립합니다.

| 마일스톤 | 목표 상세 내용 | 예상 소요 시간 | 핵심 산출물 | 안정성 기여도 |
| :--- | :--- | :--- | :--- | :--- |
| **1.1 API 엔드포인트 완성** | `/api/v1/events/summary` 및 `/api/v1/iap/transactions`의 실제 데이터 추출 로직 구현 완료. (이전 단계에서 설계된 구조 준수) | 2일 | 최종 API 핸들러 코드, 데이터 모델 정의 | 중간 |
| **1.2 $R_{Stability}$ 검증 로직 통합** | 수집된 이벤트 및 트랜잭션 데이터에 대해 시스템 안정성($R_{Stability}$) 기준을 실시간으로 필터링하고 가중치를 적용하는 백엔드 로직 구현. | 3일 | Stability Validation Service 모듈 | 높음 |
| **1.3 데이터 파이프라인 안정화** | 데이터 수집(Frontend $\to$ Backend) 과정에서 발생하는 모든 에러 및 예외 상황을 처리하는 통합 에러 핸들링 시스템 구축. | 2일 | 통합 에러 로깅/재시도 메커니즘 | 높음 |

#### 2단계: KPI 대시보드 연동 및 시각화 (Visualization & Trust)
이 단계는 확보된 안정성 데이터를 사용자에게 신뢰를 주는 형태로 제공하는 데 중점을 둡니다.

| 마일스톤 | 목표 상세 내용 | 예상 소요 시간 | 핵심 산출물 | 안정성 기여도 |
| :--- | :--- | :--- | :--- | :--- |
| **2.1 KPI 데이터베이스 설계** | $R_{Stability}$ 지표(예: 트랜잭션 성공률, 시스템 응답 시간)를 저장하고 쿼리할 수 있는 데이터베이스 스키마 최종 확정 및 마이그레이션 스크립트 작성. | 1일 | DB Schema 정의, Migration Script | 중간 |
| **2.2 대시보드 API 연동** | 1단계에서 확보한 안정성 데이터를 KPI 대시보드(Frontend)가 안전하게 요청하고 표시할 수 있도록 API 레이어 완성. | 2일 | Frontend API Endpoints | 중간 |
| **2.3 $R_{Stability}$ 기반 UI/UX 통합** | 사용자 대시보드 디자인에 시스템 안정성 지표를 핵심 요소로 통합하는 최종 UX 흐름 및 컴포넌트 구현. (Designer와의 협업 필요) | 3일 | 안정성 중심의 대시보드 프로토타입 | 높음 |
