## 🚀 API 통합 테스트 프레임워크 구축 계획 (Phase 1/2)

**목표:** $R_{Stability}$ 데이터의 정확성과 무결성을 보장하기 위한 API 통합 테스트 및 검증 환경 구축.

### Phase 1: 안정성 지표 연동 테스트 프레임워크 구축 (Focus on Data Integrity)

**핵심 목표:** 시스템 안정성($R_{Stability}$) 데이터를 제공하는 백엔드 API와 프론트엔드/분석 레이어 간의 데이터 흐름 및 인증(HTTP 401 문제 해결 포함)의 안정성을 검증합니다.

#### 1. 테스트 대상 정의 (Scope Definition)
*   **대상 API:** $R_{Stability}$ 지표를 계산하고 제공하는 모든 백엔드 엔드포인트.
*   **검증 항목:** 데이터 수집 시작(Trigger), 데이터 전송(Transmission), 데이터 처리(Processing), 사용자 접근 권한(Authentication/Authorization).

#### 2. 테스트 환경 및 도구 설정 (Setup)
*   **테스트 환경:** 스테이징(Staging) 또는 개발(Development) 환경을 기반으로, 실제 운영 환경과 유사하게 모의 데이터를 주입할 수 있는 환경 구축.
*   **도구 선정:** Python 기반 테스트 스크립트(`pytest` 권장)를 사용하여 API 호출 및 응답 검증 자동화.
    *   **검증 로직:** 데이터 정규화(Normalization) 및 통계적 지표(RMSE, MAE 등)의 일관성 확인에 중점.

#### 3. 테스트 시나리오 설계 (Scenario Design)
| 시나리오 ID | 목표 | 주요 검증 내용 | 예상 결과 ($R_{Stability}$ 관련) |
| :--- | :--- | :--- | :--- |
| **T-01** | **데이터 수집 성공** | $R_{Stability}$ 데이터가 모니터링 시스템에서 정상적으로 수집되어 API로 전송되는지 확인. | 데이터 송신 성공 (HTTP 2xx) 및 데이터 형식 일치. |
| **T-02** | **인증/권한 테스트** | $R_{Stability}$ 데이터 접근 시, 사용자 역할(Basic/Pro)에 따른 접근 권한이 정확히 적용되는지 확인. | 인증 실패(401) 또는 권한 거부(403)가 올바르게 처리됨. |
| **T-03** | **데이터 무결성 테스트** | 시스템 안정성 지표의 계산 과정에서 발생할 수 있는 데이터 누락/오류 시나리오를 주입하여 결과값의 정확성을 검증. | 예상되는 $R_{Stability}$ 값과 실제 반환값이 일치하는지 확인 (통계적 지표 검증). |
| **T-04** | **시스템 부하 테스트** | 높은 트래픽 상황에서 $R_{Stability}$ API의 응답 시간 및 안정성을 측정. | 정의된 SLA(Service Level Agreement) 내에서 응답 시간 유지. |

#### 4. 구현 및 반복 (Implementation & Iteration)
*   **구현:** 각 시나리오에 대해 자동화된 테스트 코드를 작성하고, $R_{Stability}$ 데이터의 흐름을 End-to-End로 검증합니다.
*   **반복:** 초기 테스트 실패 시, **`stability_monitor.py` 실행 오류 분석 및 API 인증 문제 해결 과정**에서 발생했던 모든 오류 패턴을 재현하여 프레임워크에 반영하고 반복합니다.

### Phase 2: Pro 티어 핵심 기능 개발 우선순위 확정 (Focus on Visualization & Reporting)

**목표:** $R_{Stability}$ 지표를 기반으로 사용자 신뢰도를 극대화하는 '안정성 시각화 및 보고서' 기능을 가장 효율적이고 안정적으로 구현합니다.

#### 1. 개발 우선순위 매트릭스
$R_{Stability}$ 달성 목표($\ge 85\%$)와 수익화 잠재력(SNR)을 기준으로 기능의 영향도와 구현 난이도를 평가하여 우선순위를 결정합니다.

| 순위 | 핵심 기능 (MVP) | $R_{Stability}$ 연관성 | 기술적 복잡도 | 개발 전략 |
| :--- | :--- | :--- | :--- | :--- |
| **P1** | **안정성 지표 시각화 대시보드 (Stability Meter)** | **최상 ($R_{Stability}$의 직접적 표현)** | 중 | 기존 KPI 대시보드 레이아웃($\text{designer.md}$)에 'Neon Vibe'를 적용하여 $R_{Stability}$ 값을 중앙에 배치하고, 실시간 변화를 강조하는 시각화 컴포넌트 구현. |
| **P2** | **안정성 보고서 (Historical Report)** | 상 (과거 추이 분석) | 중상 | 시간 경과에 따른 $R_{Stability}$의 추이(Trend)와 이상 징후(Anomaly Detection)를 시계열 데이터로 시각화하고, 주요 이벤트 발생 시점을 표시. |
| **P3** | **Pro 기능 통합 (IAP 연동)** | 중 (수익화 연결) | 중 | $R_{Stability}$ 지표 달성 여부에 따라 Pro 티어 접근 권한을 동적으로 부여하는 백엔드 로직(API Gate) 구현. |

#### 2. 기술 개발 로드맵 (P1 집중)
*   **Step 1 (Data Layer):** Phase 1에서 구축된 테스트 프레임워크를 통해 $R_{Stability}$ 데이터가 DB에 정확히 저장되고 조회되는지 최종 검증 완료.
*   **Step 2 (Visualization Core):** P1 목표인 'Stability Meter'의 핵심 컴포넌트(Neon Vibe 적용)를 구현하고, API에서 실시간으로 데이터를 받아와 동적으로 업데이트하는 프론트엔드 로직 완성.
*   **Step 3 (Reporting Layer):** 과거 데이터에 대한 분석 및 보고서 생성 모듈을 개발하여 P2 목표의 기반 마련.
