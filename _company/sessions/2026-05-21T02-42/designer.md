## 📊 KPI 시각화 대시보드 & UI/UX 요구사항 설계 (Stability Meter)

### 1. 디자인 원칙: Neon Vibe 적용
기존의 'Neon Vibe' 디자인 시스템의 핵심 요소(네온 효과, 다크 모드 기반의 대비, 생동감 있는 컬러 사용)를 $R_{Stability}$ 지표에 명확하게 적용하여 기술적 안정성을 시각적 신뢰로 전환합니다.

*   **주요 색상 팔레트:**
    *   **Primary Neon (안정성):** `#00FFFF` (밝은 시안/아쿠아) - 최대 안정 상태를 표현.
    *   **Secondary Neon (경고/주의):** `#FF00FF` (마젠타) - 임계점 도달 또는 미흡 상태를 표현.
    *   **Background:** `#121212` (깊은 블랙/차콜) - 네온 효과 극대화.
    *   **Text/Data:** `#FFFFFF` 및 은은한 라이트 그레이.

### 2. KPI 시각화 요소 설계: Stability Meter (안정성 미터)

$R_{Stability}$ 지표는 단순한 수치 표시를 넘어, 사용자가 직관적으로 현재 시스템 상태를 인지하고 신뢰하도록 만드는 것이 목표입니다.

#### A. 핵심 지표: Stability Meter
*   **형태:** 원형 또는 곡선 형태의 아날로그 미터 형태로 디자인합니다. (Neon Vibe의 유려함을 살림)
*   **작동 방식:**
    *   **0% ~ 65% (안정):** 차분한 시안(Cyan) 계열의 빛이 점진적으로 채워지며, 안정적인 상태를 표현합니다.
    *   **66% ~ 85% (주의):** 색상이 마젠타(Magenta)로 전환되며 경고 신호를 발생시킵니다.
    *   **86% ~ 95% (위험/최적화 필요):** 밝은 주황색(Amber) 또는 빨간색 계열이 혼합되어 즉각적인 주의를 환기시킵니다.
    *   **96% ~ 100% (최적):** 강렬한 네온 블루(`#00FFFF`)가 최대치로 발광하며, 시스템이 최적의 안정 상태에 있음을 시각적으로 선언합니다.

#### B. 데이터 레이아웃 및 구성 요소
| 영역 | 컴포넌트 | 요구사항 | 디자인 적용 (Neon Vibe) |
| :--- | :--- | :--- | :--- |
| **상단 요약** | $R_{Stability}$ 수치 | 현재 안정성 점수(%)를 가장 크게 표시. | 네온 블루 폰트로 중앙 배치. |
| **메인 시각화** | Stability Meter | 위에서 정의된 아날로그 미터 형태. | 배경은 어두운 블랙, 채워지는 라인은 네온 색상 그라데이션 사용. |
| **상태 피드백** | 상태 텍스트 | 현재 시스템 상태(예: '안정', '주의 필요', '최적')를 명확히 표시. | 지표 색상에 따라 폰트 색상 및 아이콘 변화. |
| **추가 정보** | 상세 분석 (Drill-down) | 안정성 저하의 원인이 되는 API 호출 실패율, Latency 데이터 등을 작은 그래프나 리스트로 제공. | 미니멀한 라인 차트와 네온 스타일의 데이터 라벨링. |

### 3. 초기 목업 방향 제시 (Wireframe Concept)

**컨셉:** **"The Stability Core"** — 시스템의 심장부(Core)가 안정적으로 빛나는 것을 표현합니다.

1.  **레이아웃:** 전체 화면은 다크 모드 배경(`#121212`)을 사용하며, 중앙에 'Stability Meter'를 배치하여 시각적 집중도를 높입니다.
2.  **상단 바 (Header):** 왼쪽에는 서비스 로고(혹은 123)와 오른쪽에는 사용자 프로필/설정 아이콘을 최소화된 네온 라인으로 표시합니다.
3.  **중앙 섹션 (The Core):**
    *   가장 큰 요소로 Stability Meter가 배치됩니다. 이 미터는 수직으로 길게 표현되어 시스템의 '성장 과정'과 '현재 상태'를 동시에 보여줍니다.
    *   미터 바 안에는 현재 안정성 %와 함께, 해당 구간에 맞는 네온 색상(Cyan $\rightarrow$ Magenta $\rightarrow$ Amber $\rightarrow$ Blue)이 동적으로 변화하며 채워져야 합니다.
4.  **하단 섹션 (Details):** Stability Meter 아래에 세 줄의 카드 형태로 상세 정보를 배치합니다.
    *   **카드 1:** 현재 상태 (`R_Stability: 92% - 최적`)
    *   **카드 2:** 주요 지표 요약 (예: API 응답 성공률, 평균 Latency)
    *   **카드 3:** 안정성 개선을 위한 제안 (예: "Pro 기능을 활성화하여 $R_{Stability}$를 95%로 목표하세요.")
