## 📊 KPI 대시보드 및 UI/UX 최종 사양 (System Stability Focus)

### 1. 핵심 목표 정의
**목표:** 시스템 안정성($R_{Stability}$) 지표를 사용자에게 투명하게 보여주어, Pro/VIP 티어 전환에 대한 신뢰도를 극대화하고 수익화를 촉진한다.

**핵심 메시지:** "안정성은 프리미엄 가치다 ($R_{Stability}$ = Trust & Premium)."

### 2. 디자인 시스템 적용: 'Neon Vibe' 통합
기존의 'Neon Vibe' 디자인 시스템을 KPI 대시보드에 적용하여 시각적 차별성을 확보합니다.

| 요소 | 세부 사양 | 적용 원칙 |
| :--- | :--- | :--- |
| **컬러 팔레트** | 주 색상: 네온 블루 ($\#00FFFF$), 강조색: 사이버 마젠타 ($\#FF00FF$), 배경: 딥 차콜 ($\#1A1A2E$) | 낮은 명도 대비를 활용하여 네온 효과 극대화. 안정성을 나타내는 녹색 계열($R_{Stability} \ge 95\%$)에만 강조색 적용. |
| **타이포그래피** | 헤더: 고정폭(Monospace) 또는 산세리프 (예: 'Space Mono', 'Orbitron'), 데이터: 깔끔한 산세리프 (예: 'Inter') | 기술적이고 미래지향적인 느낌을 부여하여 전문성을 강조. |
| **레이아웃** | 다크 모드 기반, 카드형(Card-based) 레이아웃 사용. 핵심 지표는 중앙에 배치하고, 세부 데이터는 보조 정보로 하단에 배치. | 복잡한 데이터를 시각적 위계를 통해 직관적으로 전달. |

### 3. KPI 대시보드 구조 및 구성 (Mockup 방향)

대시보드는 크게 **[상태 요약]**, **[안정성 상세 분석]**, **[수익화 연동]** 세 섹션으로 구성합니다.

#### A. 섹션 1: 상태 요약 (Stability Meter - 핵심 지표)
*   **목표:** 사용자가 한눈에 시스템의 현재 안정 상태를 인지하도록 한다.
*   **시각화:** **'Stability Meter' 원형 게이지(Gauge)** 형태를 메인으로 사용합니다.
    *   **중앙 표시:** 현재 $R_{Stability}$ 값 (예: 98.5%)을 가장 크고 네온 블루로 표시.
    *   **상태 색상 변화:**
        *   $R_{Stability} \ge 95\%$: **네온 그린** (안정 프리미엄)
        *   $90\% \le R_{Stability} < 95\%$: **앰버/오렌지** (주의 필요)
        *   $R_{Stability} < 90\%$: **딥 레드** (경고/즉각 조치 필요)
*   **보조 정보:** 현재 상태에 따른 간결한 텍스트 피드백 ("최적 안정", "주의 필요", "위험").

#### B. 섹션 2: 안정성 상세 분석 (Deep Dive Analysis)
*   **목표:** $R_{Stability}$ 수치가 어떻게 달성되었는지 구체적인 기술 지표를 제시하여 신뢰도를 구축한다.
*   **시각화:** **라인 차트(Line Chart)** 및 **히트맵(Heatmap)**을 활용합니다.
    *   **Timeline Trend:** 지난 7일/30일 동안의 $R_{Stability}$ 변화 추이를 보여주는 라인 차트. (Neon Vibe의 빛줄기 효과 적용)
    *   **Component Breakdown:** 시스템의 주요 모듈(API Latency, Error Rate, Memory Usage 등)별로 안정성 기여도를 나타내는 작은 바 그래프 또는 히트맵을 제공.

#### C. 섹션 3: 수익화 연동 (Monetization Link)
*   **목표:** 안정성이 직접적으로 IAP 가치와 연결됨을 명확히 한다.
*   **시각화:** **티어별 안정성 비교 테이블** 및 **전환 로직 시각화**.
    *   **테이블 예시:** Basic / Pro / VIP 티어별 요구 $R_{Stability}$ 목표를 비교하여 표시. (Pro 전환 시 필요한 안정성 기준을 강조)
    *   **CTA 연동:** 'Pro 업그레이드' 버튼 옆에 "안정성 보장: $98\%$ 달성"과 같은 문구를 노출하여 심리적 장벽을 낮춘다.

### 4. 개발팀 전달용 최종 요구사항 (Actionable Spec)

1.  **데이터 소스 정의:** 모든 KPI는 API `/api/stability/report`에서 실시간으로 가져와야 하며, $R_{Stability}$ 지표가 IAP 티어별 잠금 해제 조건과 직접적으로 매핑되어야 함을 명시.
2.  **디자인 시스템 준수:** 전체 대시보드는 'Neon Vibe'의 컬러 팔레트($\#00FFFF$, $\#FF00FF$)를 핵심 액센트로 사용하고, 다크 모드 배경을 유지해야 함.
3.  **상태 기반 인터랙션:** $R_{Stability}$ 값 변화에 따라 게이지 색상이 실시간으로 동적 변경되어야 하며, 경고 상태($<95\%$) 시에는 즉각적인 알림(Notification)이 발생하도록 구현.
4.  **UI/UX 흐름 (Monetization Flow):** 사용자가 Pro 티어 전환을 고려할 때, '안정성 확보'가 가장 중요한 구매 동인임을 명확히 제시하는 시각적 흐름(Section 3)을 최우선으로 설계해야 함.
