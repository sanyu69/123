## 📊 시스템 안정성($R_{Stability}$) 기반 KPI 대시보드 설계 사양

이 대시보드는 기술적 안정성을 비즈니스 성과(수익화, 사용자 유지)로 연결하는 것을 목표로 합니다.

### 1. 핵심 측정 지표 정의 (KPIs for $R_{Stability}$)

시스템 안정성($R_{Stability}$)은 크게 **기술적 안정성**과 **사용자 경험 안정성**으로 나누어 측정하며, 이는 최종적으로 수익화 모델에 반영되어야 합니다.

| 분류 | KPI 항목 | 측정 기준 및 목표 | 비즈니스 연관성 |
| :--- | :--- | :--- | :--- |
| **기술적 안정성 (Technical)** | **오류율 (Error Rate)** | 5xx 서버 오류 발생 빈도 (목표: 0.1% 미만) | 사용자 이탈 방지, 신뢰도 구축 |
| | **로드 시간 (Load Time)** | 핵심 게임 로딩 시간 (P95 기준): 목표 3초 이하 | 사용자 경험(UX), 즉각적인 만족도 |
| | **API 응답 지연 (Latency)** | 주요 IAP 결제/데이터 요청의 평균 응답 시간 (목표: 500ms 이하) | 거래 안정성, 인앱 구매 성공률 |
| | **코드 무결성 점수** | 자동화된 테스트 통과율 및 코드 리뷰 오류 수 (목표: 100% 통과) | 개발 비용 절감, 유지보수 효율성 |
| **사용자 경험 안정성 (UX Stability)** | **세션 이탈률 (Session Drop Rate)** | 게임 플레이 중 연결 끊김 또는 기능 미작동으로 인한 세션 종료 비율 (목표: 1% 미만) | 사용자 유지율(Retention), 몰입도 측정 |
| | **결제 성공률 (Conversion Rate)** | IAP 구매 시도 대비 실제 거래 완료 비율 (목표: 98% 이상) | 수익화 모델의 직접적 성과 |
| | **오류 보고 빈도** | 사용자 측에서 보고된 버그/오류 티켓 수 (목표: 주당 1건 미만) | 고객 지원 비용 절감, 신뢰 확보 |

### 2. 대시보드 구조 설계 (Dashboard Structure)

대시보드는 '상단 요약'과 '세부 분석'으로 구성되어야 하며, $R_{Stability}$ 지표가 최우선적으로 시각화되어야 합니다.

**A. 상단 요약 섹션 (Executive Summary)**
*   **핵심 안정성 점수:** 종합 $R_{Stability}$ 점수를 100점 만점으로 표시 (기술적 안정성과 UX 안정성의 가중 평균)
*   **주요 지표 카드:** 오류율(%), 평균 로딩 시간(s), 결제 성공률(%)을 가장 큰 숫자로 표시.
*   **경고/알림:** 실시간으로 위험 수준(예: 오류율 1% 초과 시 빨간색 경고)을 표시.

**B. 세부 분석 섹션 (Deep Dive Analytics)**
1.  **기술 안정성 트렌드:** 시간 흐름에 따른 오류율, 로딩 속도 추이 그래프 (장기적인 개선 추세 파악).
2.  **수익화 안정성:** 결제 성공률과 이탈률의 상관관계 분석 차트. (특정 불안정성이 수익에 미치는 영향 분석)
3.  **사용자 피드백 통합:** 오류 보고 빈도 및 사용자 리뷰(버그 관련 언급)를 시계열로 표시하여, 사용자의 실제 경험 기반으로 안정성을 평가.

### 3. 데이터 소스 및 연동 전략

*   **데이터 소스:** 웹사이트 프론트엔드 로그 (Performance API), 백엔드 서버 에러 로깅 시스템 (Error Log DB), 결제 게이트웨이(PayPal) 트랜잭션 데이터.
*   **연동 목표:** 모든 데이터는 실시간 또는 일간 단위로 집계되어야 하며, **개발팀과 비즈니스팀이 동일한 안정성 정의를 공유**하도록 합니다.
