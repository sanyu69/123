## 💎 Reliable Neon UI/UX 명세서 (Code-Ready Specification)

### 1. 디자인 시스템 정의 ($R_{Stability}$ 기반)

**핵심 원칙:** 모든 시각 요소는 'Neon\_Vibe\_Palette'를 따르되, 정보의 계층 구조(Hierarchy)와 상호작용(Interaction)이 명확하여 사용자가 안정적으로 경로를 탐색할 수 있도록 설계합니다.

| 속성 | 값/규칙 | $R_{Stability}$ 연관성 |
| :--- | :--- | :--- |
| **Primary Color** | `#00FFFF` (Neon Cyan) | 주 기능 및 성공 상태 강조 |
| **Secondary Color** | `#FF00FF` (Neon Magenta) | 보조 액션 및 경고/주의 상태 강조 |
| **Background** | `#1A1A2E` (Deep Navy/Dark Purple) | 대비 극대화, 시각적 안정성 확보 |
| **Accent/Text** | `#FFFFFF` 또는 `#00FF00` (Neon Green) | 텍스트 가독성 및 핵심 정보 강조 |
| **Font Family** | `Orbitron`, `Space Mono` (혹은 시스템 기본 Sans-serif) | 네온/미래지향적 분위기 유지 |
| **Spacing Unit** | `8px` 기반의 배수 사용 (모든 간격은 8의 배수로 설정) | 일관성 및 그리드 안정성 확보 |

### 2. 페이지 구조 정의 (Layout Structure)

웹사이트는 세 가지 주요 섹션으로 구성되며, 각 섹션은 명확한 시각적 분리(Separation)를 통해 정보 과부하를 방지합니다.

**A. Header/Navigation (상단 내비게이션)**
*   **레이아웃:** 고정형(Sticky). 좌측에 로고/브랜드명, 우측에 주요 메뉴 및 IAP CTA 버튼 배치.
*   **스타일:** 배경색은 다크 모드(`Background`)를 유지하며, 활성 메뉴는 `Primary Color`로 하이라이트 처리.

**B. Main Content Area (메인 콘텐츠 영역)**
*   **레이아웃:** 2단 그리드 시스템을 채택하여 정보의 안정적인 배치를 보장합니다.
    *   **Left Panel (60%):** 제품 설명, 기능 소개, $R_{Stability}$ 강조 메시지 배치.
    *   **Right Panel (40%):** 핵심 시각 자료(목업), 가격 플랜 비교표, 주요 CTA 버튼 배치.

**C. Footer (하단 영역)**
*   **레이아웃:** 단순 정보 제공 및 법적 고지 사항.
*   **스타일:** 최소화된 디자인으로 시각적 방해 요소를 제거하여 안정성을 높입니다.

### 3. 핵심 컴포넌트 정의 (Component Definition)

모든 컴포넌트는 위에서 정의한 색상 팔레트와 간격 규칙을 엄격하게 준수해야 합니다.

#### 3.1. Pricing Plan Card (가격 플랜 카드)
*   **목표:** IAP 티어(Basic/Pro/VIP)를 명확하게 구분하고, $R_{Stability}$에 따라 프리미엄 가치를 시각적으로 전달합니다.
*   **구조:** 각 티어는 독립된 카드로 구성되며, 선택된 티어는 `Primary Color` 테두리와 배경 하이라이트로 강조됩니다.
    *   **Basic:** 표준 레이아웃. 기능 목록은 간결하게 표시.
    *   **Pro (Recommended):** 카드 전체에 `Secondary Color`를 미묘하게 적용하여 프리미엄 느낌을 부여하고, '추천' 배지를 명확히 삽입합니다.
    *   **VIP:** 가장 강력한 `Primary Color` 강조 및 독점 기능 목록을 배치하여 최고 가치를 시각적으로 표현합니다.

#### 3.2. Feature List Component (기능 리스트)
*   **목표:** 시스템의 안정성과 핵심 기능을 명확하게 전달합니다.
*   **구조:** 각 기능 항목은 아이콘(Neon 스타일) + 제목 + 설명으로 구성됩니다.
    *   **안정성 표시:** 모든 주요 기능 옆에는 작은 '✅' 또는 '⚡' 아이콘을 배치하여 기능의 확실성과 안정성을 시각적으로 보장합니다.

#### 3.3. CTA Button Component (Call to Action 버튼)
*   **목표:** 사용자의 다음 행동을 명확하게 유도합니다.
*   **스타일:** 버튼은 배경색(`Primary Color` 또는 `Secondary Color`)으로 채워지며, 마우스 오버 시 색상 반전(Inversion) 효과를 적용하여 반응성을 높입니다. 폰트는 산세리프 계열을 사용하여 명료함을 유지합니다.

### 4. 코다리를 위한 최종 지침 요약

코다리는 위 명세서를 기반으로 HTML/CSS에 다음과 같이 구현해야 합니다:
1.  **색상 및 타이포그래피 적용:** 정의된 `Neon_Vibe_Palette`와 폰트 규칙을 CSS 변수 또는 클래스로 정의하여 모든 요소에 일관되게 적용한다.
2.  **그리드 시스템 확보:** 모든 레이아웃은 `8px` 간격 기반의 그리드(Grid) 시스템 내에서 움직이도록 구현하여 $R_{Stability}$를 충족시킨다.
3.  **컴포넌트 분리:** Pricing Card, Feature List 등은 독립적인 컴포넌트로 설계되어야 하며, 각 컴포넌트는 상태(State)에 따라 시각적 변화가 명확하게 정의되어야 한다.
