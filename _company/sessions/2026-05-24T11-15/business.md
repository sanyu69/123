### 📝 개발 착수를 위한 최종 요구사항 (Action Items)

개발팀이 즉시 착수하기 위해 다음 세 가지 핵심 요소를 반드시 확정해야 합니다. 이 항목들은 **API 명세서(`vip_pro_api_spec.md`)**에 반영되어야 합니다.

1.  **데이터-UI 매핑 테이블 정의:**
    *   각 API 엔드포인트(예: `/api/v1/user/stability`)에서 반환되는 데이터 필드($R_{Stability}$ 점수, 구독 상태 등)가 Designer가 요구한 시각적 요소(VIP Highlight 색상 적용 여부, 배경색 등)에 **어떻게 직접적으로 매핑**되어야 하는지에 대한 명확한 JSON 스키마 정의를 추가해야 합니다.

2.  **$R_{Stability}$ 지표의 백엔드 로직 최종 확정:**
    *   `stability_logic.py`에서 계산된 $R_{Stability}$ 값이 어떤 티어(VIP/Pro/Basic)에 따라 **실제 API 응답 값으로 변환**되는지, 그리고 이 값이 프론트엔드의 시각적 계층 구조(`Dashboard Prototype`)와 완벽하게 일치하도록 백엔드 로직의 최종 계산 기준을 명시해야 합니다.

3.  **최종 UI/UX 검증 승인:**
    *   Designer는 제작된 프로토타입이 모든 IAP 티어(Basic, Pro, VIP)에 대해 **예상되는 데이터 상태**를 완벽히 반영하고 있는지 확인한 후, 이 시각적 요구사항을 API 명세서에 공식적으로 승인해야 합니다.
