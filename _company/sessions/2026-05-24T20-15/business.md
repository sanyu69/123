# 🚀 기능 게이팅 및 A/B 테스트 계획 (KPI & Data Strategy)

## 1. 핵심 KPI 정의: '안정성 기반의 전환율' 측정

기능 게이팅 시스템의 성공은 단순히 결제 수입을 넘어, 사용자가 **지불하는 프리미엄 가격에 상응하는 안정성을 경험**했는지 측정해야 합니다. 따라서 KPI는 '전환율'과 '체류 시간/이탈률'의 복합 지표로 정의합니다.

### A. 핵심 성공 지표 (Primary KPIs)

1.  **Tier Conversion Rate ($CR_{Tier}$):**
    *   **정의:** 특정 티어(VIP 또는 Pro)를 구독하는 사용자 비율.
    *   **목표:** $CR_{VIP} > X\%$ (예: 5%) 및 $CR_{Pro} > Y\%$ (예: 15%).
2.  **Stability-Driven Retention ($R_{Stability\_Retention}$):**
    *   **정의:** 특정 안정성 임계값($R_{Stability}$) 이상을 유지하는 사용자의 월간 잔존율.
    *   **목표:** VIP/Pro 그룹에서 $R_{Stability} \ge 0.8$인 사용자의 3개월 Retention Rate를 전체 평균보다 최소 20% 높게 유지. (안정성이 곧 지속적인 가치로 이어짐을 증명)
3.  **Feature Utilization Rate ($U_{Feature}$):**
    *   **정의:** 각 티어에 독점적으로 제공되는 핵심 기능(예: 고급 데이터 시각화, 무제한 API 호출 등)을 활발하게 사용하는 사용자 비율.
    *   **목표:** Pro/VIP 사용자가 해당 기능 중 최소 80% 이상을 활용하도록 유도하여 가격 대비 가치를 증명.

### B. 보조 지표 (Secondary Metrics)

1.  **Churn Rate by Tier ($CR_{Churn}$):** 각 티어별 월간 해지율. (가장 직접적인 수익성 지표)
2.  **Latency Impact Score ($L_{Impact}$):** API 응답 지연 시간(Latency)이 사용자 이탈에 미치는 상관관계 분석.

## 2. A/B 테스트 설계: $R_{Stability}$와 가격 민감도 검증

A/B 테스트는 **가격 민감도**와 **안정성 인식의 차이**를 검증하는 데 초점을 맞춥니다.

### A. 테스트 가설 (Hypotheses)

*   **가설 1 (가격 민감도):** 더 높은 가격($9,900원$ vs $4,900원$)은 사용자의 '안정성' 인식을 높여, 낮은 안정성 사용자에게는 더 강력한 프리미엄으로 인식될 것이다.
*   **가설 2 (안정성 임계값):** $R_{Stability}$의 특정 임계값($R_{Threshold}$)을 설정했을 때, 사용자의 전환율이 극대화될 것이다.

### B. 테스트 그룹 및 변수 설계

| 그룹 | 가격 구조 | 핵심 실험 변수 | 측정 목표 |
| :--- | :--- | :--- | :--- |
| **Control (A)** | 현행 ($9,900/4,900$) | 표준 $R_{Stability}$ 기반 게이팅 | 기준선 성능 설정 |
| **Test 1 (B)** | 가격 조정 (VIP: $12,900 / Pro: $6,900$) | 가격 민감도 검증 | 가격 상승에 따른 전환율 변화 분석 |
| **Test 2 (C)** | 안정성 임계값 변경 ($R_{Threshold}$ 상향) | 프리미엄 인식 강화 | $R_{Stability} \ge 0.9$ 사용자 그룹의 유지율 측정 |

### C. 데이터 수집 계획

1.  **데이터 소스:** `stability_logic.py`에서 실시간으로 계산되는 $R_{Stability}$ 값, 사용자 행동 로그(클릭, 세션 시간), 결제 기록 데이터.
2.  **수집 주기:** 주간 단위로 집계하여 추세 분석을 진행하고, 월간 단위로 최종 KPI를 산출합니다.
3.  **실행 방법:** 코다리 에이전트가 $R_{Stability}$ 데이터를 기반으로 A/B 그룹별 전환율과 Retention Rate를 계산하는 보고서를 생성하도록 요청합니다.
