### 2단계: Designer 전달용 데이터 구조 스펙 정의

Designer가 '네온 안정성 강조' 컨셉에 맞춰 VIP/Pro 대시보드 에셋을 제작할 수 있도록, 기술적 안정성 지표($R_{Stability}$)를 시각적으로 가장 잘 표현할 수 있는 최소한의 데이터 구조를 정의합니다.

**데이터 스펙 명칭:** `StabilityMetricPayload` (VIP/Pro 티어별)

이 스펙은 API 응답으로 사용될 것이며, 시각적 강조에 초점을 맞춥니다.

```json
{
  "tier": "Pro", 
  "stability_score": {
    "R_Stability_Value": 0.95, // 실제 측정된 안정성 지표 (0.0 ~ 1.0 사이 값)
    "stability_trend": "increasing", // 최근 데이터 대비 추세 (increasing, decreasing, stable)
    "stability_level_text": "High Stability (Neon)", // 시각적 강조를 위한 최종 레이블
    "risk_indicator": {
      "latency_ms": 45,          // API 응답 지연 시간 (ms). 낮을수록 좋음.
      "error_rate_percent": 0.1, // 시스템 오류 발생률 (%). 낮을수록 좋음.
      "uptime_hours": 24.5       // 시스템 가동 시간 (시간 단위)
    }
  },
  "pricing_context": {
    "monthly_cost_krw": 4900, // Pro 티어 월 비용 (원화 기준)
    "stability_value_per_cost": "High Value" // 이 가격에 대한 안정성 가치 평가
  }
}
```

**스펙 설명:**

1.  **`tier`**: 현재 적용된 구독 등급 (`Basic`, `Pro`, `VIP`). 디자인 구분에 필수적입니다.
2.  **`stability_score`**: 핵심 지표로, $R_{Stability}$ 값을 0.0에서 1.0 사이의 정규화된 값으로 변환하여 시각적 게이지나 네온 효과의 강도로 직접 매핑합니다.
3.  **`stability_trend`**: 단순한 수치보다 추세를 강조하여 사용자가 시스템의 '상태'를 직관적으로 파악하게 합니다. (예: 녹색/빨간색 톤을 동적으로 변경하는 데 활용)
4.  **`risk_indicator`**: $R_{Stability}$가 내부적으로 측정하는 세부 요소들(지연 시간, 오류율, 가동 시간)을 제공하여, '안정성'이 단순히 하나의 숫자가 아닌 여러 요소의 조합임을 시각화합니다.
5.  **`pricing_context`**: 프리미엄 가격($4,900원/월, $9,900원/월)과 안정성 가치($Stability\_Value$)를 연결하여, 사용자가 지불하는 금액이 **안정성에 대한 투자**임을 명확히 전달합니다.
