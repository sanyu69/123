이 자료는 현빈 에이전트의 IAP 가격 전략 시뮬레이션을 위한 필수 입력 항목들입니다.

### 📊 전환율 시뮬레이션 모델 입력 요구사항 요약

**1. 핵심 목표 변수 (Input Variables)**

*   **$S_{Success}$ (PayPal 성공률):** 초기 테스트 기간 동안 확보된 실제 결제 성공률 데이터. (목표치: $98\%$)
*   **$R_{Stability}$ (시스템 안정성):** 서버 응답 시간의 변동성 지표. (이 변수가 전환율에 미치는 영향 분석 필요)
*   **$\Delta T$ (작업 지연 시간):** 아트 에셋 완료 시점과 코드 적용 가능 시점 간의 지연 시간 데이터.

**2. 가격 전략 및 기능 변수 (Pricing & Feature Variables)**

*   **티어별 가격 ($P_{VIP}, P_{Pro}$):** A안($4.99, $9.99) 및 B안에 대한 구체적인 금액 설정값.
*   **가치 차별화 변수 ($\text{Differentiator Score}$):** Cyan vs Magenta 비주얼 차별화를 통해 사용자 경험을 얼마나 극대화했는지 정량화할 수 있는 지표. (디자이너 산출물 기반)
*   **기능 세트 ($F_{VIP}, F_{Pro}$):** 각 티어에 포함된 구체적인 기능 목록 및 범위.

**3. 시뮬레이션 모델 포맷 (Simulation Format)**

다음과 같은 구조로 데이터를 입력하여 전환율($S_{Success}$) 예측을 수행할 수 있습니다.

*   **입력 포맷:** CSV 또는 JSON 형태를 권장합니다.
*   **핵심 컬럼:**
    1.  **시나리오 ID:** (예: A안, B안)
    2.  **가격 설정 ($P$):** 해당 시나리오의 VIP/Pro 가격.
    3.  **경험 변수 ($\text{Exp}$):** 비주얼 차별화 수준 또는 기능 제공 범위 점수.
    4.  **예측 성공률 ($S_{Predicted}$):** $S_{Success}$ 데이터를 기반으로 예측된 전환율 값.
    5.  **안정성 조정 계수 ($\alpha$):** $R_{Stability}$ 지표를 반영한 안정성 보정 계수.

**4. 다음 단계 제안**

이 포맷을 바탕으로, 현빈 에이전트가 확보한 $S_{Success}$ 데이터를 입력하여 **보수적/공격적 IAP 옵션 A/B 시나리오의 실제 전환율($S_{\text{Actual}}$) 예측 모델** 구축에 착수할 것을 제안합니다.
