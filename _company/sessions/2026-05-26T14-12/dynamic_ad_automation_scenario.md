# 동적 광고 예산 조정 시나리오 (Phase 2)

## 🎯 목표: 환경 변수 기반의 실시간 재정 통제 시스템 구축

### 1. 트리거 및 상태 정의
*   **트리거**: 코다리/영숙 에이전트로부터의 환경 변수 수신 상태($T_{Env\_Status}$).
*   **상태 A (Pending)**: 데이터 수신 지연 발생 시. $\rightarrow$ 광고 집행 **Freeze**.
*   **상태 B (Acquired)**: 모든 API 정보 확보 완료 시. $\rightarrow$ 광고 집행 **Scale Up**.

### 2. 수익화 연동 전략 (VIP/Pro 기반)
수익화 목표는 단순 노출이 아닌, 사용자 전환에 집중한다.

| 상태 | 광고 예산 조정 | 핵심 KPI 초점 | 행동 규칙 |
| :--- | :--- | :--- | :--- |
| **Pending** | 0% (Freeze) | 콘텐츠 준비 및 데이터 확보. | 대기 모드: 리소스 재분배. |
| **Acquired** | 100% (Scale Up) | VIP/Pro 티어 전환율(CVR), AOV, ROI. | 실행 모드: 실시간 예산 자동 조정. |

### 3. 동적 실행 로직 상세
1.  **모니터링**: $T_{Env\_Status}$를 실시간으로 폴링한다.
2.  **Pending 시**: 광고 집행 모듈을 OFF하고, Designer/Writer의 콘텐츠 준비에 리소스를 집중시킨다. (데이터 확보 전 '준비' 단계 강화)
3.  **Acquired 시**: VIP/Pro 티어 전환 목표를 기준으로 예산을 분배한다.
    *   $E_{VIP}, E_{Pro}$ 할당 시, $ROI \ge 2.5$ 조건을 필터로 적용하여 비효율적인 집행을 방지한다.

### 4. 핵심 ROI 설계
*   **핵심 지표**: VIP/Pro 티어의 $\text{ROAS} = \frac{\text{순수 수익}}{\text{광고 비용}}$를 최우선으로 추적한다.
*   **자동 조정 규칙**:
    *   $ROI < 2.5$: 광고 입찰을 하향하고 콘텐츠 품질 개선에 예산을 재투자한다. (비효율성 제거)
    *   $ROI \ge 3.0$: 현재 규모를 유지하거나, 다음 단계의 수익화 확장(예: 프리미엄 기능 추가)을 위한 테스트 예산을 할당한다.
