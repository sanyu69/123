### 1. $R_{Stability}$ 기반 E2E 테스트 자동화 우선순위 재확정

기존 로드맵에 따라, 시스템 안정성($R_{Stability}$) 확보를 위한 작업은 단순한 기능 구현보다 **데이터 파이프라인의 무결성과 API 연동의 안정성**에 집중해야 합니다.

| 순위 | 마일스톤 명 | 목표 범위 | $R_{Stability}$ 기여도 | 실행 우선순위 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | **데이터 무결성 E2E 검증** | `stability_monitor.py` 결과 $\rightarrow$ DB/API 반영까지의 전체 파이프라인 테스트 | **최상 (핵심)** | **즉시 착수** |
| **2** | **인증 및 트랜잭션 안정화** | 사용자 결제(IAP) 흐름, API 인증(HTTP 401 문제 해결)의 End-to-End 검증 | 높음 | M1 완료 직후 병행 |
| **3** | **프론트엔드 상태 동기화** | 백엔드 데이터 변화가 프론트엔드 UI에 정확히 반영되는지 확인 (State Management 안정성) | 중간 | 2차 개발 단계에서 통합 |

**결론:** 웹사이트 완성의 병목은 '게임 로직 구현' 자체가 아니라, **안정적인 데이터 흐름과 사용자 결제/인증 프로세스를 보장하는 백엔드-프론트엔드 간의 신뢰성 확보**에 있습니다. 따라서 E2E 테스트는 기능 테스트보다 이 안정화 작업에 80%의 자원을 할애해야 합니다.

### 2. 웹사이트 완성의 기술적 병목 지점 및 해결 방안

웹사이트 완성(게임 웹사이트)을 위한 주요 기술적 병목은 다음과 같습니다.

#### A. 병목 지점: 데이터 안정성 (Backend/Data Layer)
*   **문제:** $R_{Stability}$ 측정 모듈에서 확보된 데이터를 API 엔드포인트로 전달하는 과정에서의 인증 문제($HTTP 401$) 및 데이터 무결성 확보가 최종 사용자 경험에 직접적인 영향을 미칩니다.
*   **해결 방안:**
    1.  **인증/인가 계층 분리:** $R_{Stability}$ 데이터 송신을 위한 별도의, 권한이 제한된 API 엔드포인트와 인증 토큰 체계를 구축하여, 핵심 게임 로직과 결제 시스템의 보안 경계를 명확히 분리합니다.
    2.  **트랜잭션 격리 강화:** $R_{Stability}$ 데이터가 DB에 기록되는 시점과 외부 서비스(API)로 전송되는 시점을 원자성(Atomicity)이 보장되도록 트랜잭션 레벨에서 명확히 묶어 처리합니다.

#### B. 병목 지점: 프론트엔드 상태 관리 (Frontend/UI Layer)
*   **문제:** 백엔드의 불안정성이나 데이터 업데이트 지연이 사용자 인터페이스(게임 화면, 보상 표시 등)의 일관성을 해쳐 사용자 신뢰도를 저하시킵니다.
*   **해결 방안:**
    1.  **단방향 데이터 흐름 (Flux/Redux 패턴 강화):** 모든 게임 상태와 $R_{Stability}$ 지표는 중앙 집중식 상태 관리(예: Redux, Zustand)를 통해 관리하며, 서버로부터 받은 데이터만 반영하도록 강제합니다.
    2.  **낙관적 업데이트 전략:** 사용자 입력에 대한 즉각적인 피드백을 위해, 로컬에서 먼저 상태를 업데이트한 후 백엔드 검증(Validation)을 거쳐 최종적으로 상태를 확정하는 방식으로 UX 지연 시간을 최소화합니다.

#### C. 병목 지점: 배포 및 환경 일관성 (DevOps Layer)
*   **문제:** 개발, 테스트, 프로덕션 환경 간의 차이로 인해 발생하는 예기치 않은 오류(환경 설정 불일치, 의존성 문제)는 $R_{Stability}$를 훼손합니다.
*   **해결 방안:**
    1.  **Docker 기반 환경 통일:** 모든 서비스(API 서버, 데이터베이스, 모니터링 에이전트)를 Docker 컨테이너로 패키징하여, 개발 환경과 운영 환경의 **환경 일관성(Reproducibility)**을 100% 보장합니다.
    2.  **CI/CD 파이프라인 자동화:** E2E 테스트($M1$)가 통과될 때만 배포가 진행되도록 CI/CD 파이프라인을 설정하여, 수동 개입으로 인한 오류 가능성을 제거합니다.
