# 💰 IAP 및 수익화 기능 요구사항 명세 (스펙)

**목표:** $R_{Stability}$를 핵심 가치로 삼아, 사용자에게 안정적인 경험과 프리미엄 기능을 제공하고, 옵션 B 가격 정책을 통해 최대의 ROI를 달성한다.

## 1. IAP 티어 구조 및 기능 매핑

| 티어 | 목표 수익 (옵션 B 기반) | 핵심 기능 요구사항 (Functional Requirements) | $R_{Stability}$ 연관성 |
| :--- | :--- | :--- | :--- |
| **Basic** | 무료/진입점 확보 | 핵심 게임 플레이 접근 권한, 광고 없음. 제한된 인앱 구매(IAP) 기능 허용. | 기본 시스템 안정성 유지 (필수). |
| **Pro** | 중급 사용자 유치 | Basic 기능 + 고급 게임 내 커스터마이징 옵션 활성화, 광고 제거, 데이터 저장 용량 증가. | 사용 경험의 질 향상 및 안정적 운영 기반 제공. |
| **VIP** | 최고 가치 사용자 확보 | Pro 기능 + 독점 아트 에셋 접근 권한, 실시간 시스템 안정성 지표(미시적) 대시보드 연동 (선택), 광고 완벽 제거. | 최고 수준의 시스템 안정성과 독점 경험 제공에 대한 보상. |

## 2. 수익화 흐름 (Monetization Flow) 상세 스펙

**핵심 목표:** 인앱 구매(IAP)가 매끄럽고, 서버 측 결제 모듈(`payment_module.py` 기반)을 통해 $R_{Stability}$를 증명하는 흐름을 구축한다.

### 2.1. 결제 및 인증 흐름 (Payment & Auth Flow)

*   **요구사항:** 모든 IAP 구매는 `payment_flow_design.md`에 명시된 서버 측 로직(`payment_module.py`)을 통해 처리되어야 한다.
*   **흐름 상세:**
    1.  사용자가 특정 티어(Basic/Pro/VIP) 선택.
    2.  프론트엔드는 사용자 인증 후, 서버에 결제 요청 (IAP ID 및 티어 정보 포함).
    3.  서버는 `paypal_revenue` 도구 또는 자체 로직을 통해 유효성을 검증하고 결제를 처리한다.
    4.  결제 성공 시, 데이터베이스 내 사용자 계정의 권한 레벨(`tier`)을 즉시 업데이트한다.
    5.  **안정성 체크:** 모든 IAP 트랜잭션은 $R_{Stability}$ 지표에 영향을 주지 않도록 비동기적으로 처리되어야 한다.

### 2.2. 기능 잠금 및 접근 제어 (Feature Gating)

*   **요구사항:** 모든 프리미엄 기능(Pro, VIP)은 서버 측에서 사용자 티어(`tier`)를 검증한 후에만 활성화된다. 클라이언트 측 조작을 통한 우회 시도를 방지한다.
*   **구현 상세:**
    *   게임 내 모든 리소스 로딩 및 UI 컴포넌트 렌더링 시, 현재 사용자의 `tier` 정보를 서버에서 받아와 권한을 확인하는 API 호출이 필수적이다.
    *   예: VIP 전용 아트 에셋은 `is_vip = True`일 경우에만 로드된다.

## 3. 시스템 안정성($R_{Stability}$) 연계 스펙

**목표:** 수익화 모델(옵션 B)의 성공을 $R_{Stability}$ 지표와 직접적으로 연결한다.

*   **KPI 설계:**
    1.  **$R_{Stability}$ 측정 지표 정의:** 서버 응답 시간, 결제 실패율, 세션 유지 시간 등을 핵심 지표로 설정한다. (이 데이터는 `payment_module.py` 실행 결과 및 시스템 로그에서 추출되어야 함).
    2.  **수익화 목표 연계:** 특정 IAP 티어(특히 VIP)의 사용자 그룹이 다른 티어 대비 $R_{Stability}$ 측면에서 통계적으로 우월한지 분석하는 대시보드를 구축한다. (이는 향후 KPI 대시보드 운영 목표가 됨).
*   **기술 요구사항:**
    *   모든 서버 엔드포인트는 최소 응답 시간(Latency) 기준을 충족해야 하며, 이는 $R_{Stability}$의 하위 지표로 간주한다.
    *   결제 실패율은 $R_{Stability}$에 직접적인 부정적 영향을 미치는 핵심 위험 요소로 관리되어야 한다.
