CEO의 지시사항에 따라 $C_{min}$ 기준을 기반으로 웹사이트에서 가장 안정성을 저해하는 시각적 요소를 식별하고, 이를 최소화하기 위한 구체적인 인터랙션 요구사항($R_{Stability}$-Compliant Interaction Specs)을 정의합니다.

## 1. 안정성을 저해하는 핵심 시각적 요소 식별

$C_{min}$ 기준(Latency P95 < 200ms, Error Rate < 0.1%) 하에서 사용자의 인지 부하를 최대화하고 잠재적인 시스템 불안정성을 시각적으로 노출하는 요소는 **불확실한 로딩 상태**와 **과도한 애니메이션의 오용**입니다.

*   **가장 위험한 요소:** **모호한 로딩 상태 (Ambiguous Loading States)**
    *   사용자는 응답이 없는 상태를 '시스템 오류' 또는 '느린 연결'로 해석하며, 이는 $R_{Stability}$에 대한 신뢰도를 즉각적으로 훼손합니다. 특히 데이터 로드 중 발생하는 지연은 사용자가 다음 액션을 취하기 전에 이탈을 유발하는 주범입니다.
*   **차선 위험 요소:** **과도한 시각적 피드백 (Excessive Visual Feedback)**
    *   불필요하게 복잡하거나 지속적인 애니메이션(예: 모든 요소가 끊임없이 깜빡이거나 변형되는 것)은 실제 시스템 응답 속도를 가리고, 사용자가 시스템의 상태를 정확히 파악하는 것을 방해하여 불안감을 증폭시킵니다.

## 2. $R_{Stability}$-Compliant Interaction Specs 정의

위에서 식별된 위험 요소를 최소화하기 위해, **기술적 상태($C_{min}$)를 시각적으로 투명하게 전달**하고 **사용자에게 예측 가능한 경험**을 제공하는 인터랙션 명세서를 다음과 같이 정의합니다.

### A. 로딩 상태 관리 (Loading State Management)

목표: 지연 발생 시 시스템 오류가 아닌, '처리 중'임을 명확히 인지시키고 사용자 이탈을 방지한다.

| 상태 구분 | 기술적 조건 ($R_{Stability}$ 연관) | 시각적 표현 (Visual Spec) | 상호작용 요구사항 (Interaction Spec) |
| :--- | :--- | :--- | :--- |
| **초기 로드 (Initial Load)** | API 호출 시작. Latency 측정 시작. | **Skeleton Loader (뼈대 로딩):** 콘텐츠의 레이아웃만 먼저 표시하고, 실제 데이터 영역은 회색/흰색 블록으로 채움. | 즉시 반응성(Perceived Performance)을 높이기 위해 최소한의 CSS 트랜지션(0.1s 미만)만 적용한다. |
| **데이터 처리 중 (Processing)** | 서버 응답 대기 또는 클라이언트 측 데이터 변환 중. | **Subtle Pulsing Indicator:** 로딩 인디케이터는 고정된 형태를 유지하되, 미세하게(50ms 주기) 내부 색상이 흐릿하게 맥동(Pulsing)하도록 한다. | 애니메이션은 부드러워야 하며, 사용자가 '멈춤'을 기다리는 느낌이 아닌 '시스템이 작업 중'임을 인지시키는 데 집중한다. **절대 깜빡임 금지.** |
| **에러 발생 (Error State)** | Error Rate > 0.1% 또는 Latency 초과 ($C_{min}$ 위반). | **Contextual Error Banner:** 화면 전체를 가리지 않는, 해당 요소 근처에 명확한 오류 메시지를 표시한다. 색상은 경고(Warning Red)로 설정한다. | 사용자가 에러의 원인(API, 네트워크 등)을 즉시 파악할 수 있도록, 액션 가능한 버튼(Retry/Refresh)을 반드시 제공한다. |
| **성공 상태 (Success State)** | $C_{min}$ 기준 충족. 데이터 렌더링 완료. | **Smooth Transition:** 로딩 상태에서 최종 콘텐츠로 전환 시, 부드러운 페이드 인(Fade-in) 또는 슬라이드 효과를 적용한다. | 애니메이션은 최소화하고, 정보 전달의 명확성에 우선순위를 둔다. (Latency가 낮을 경우, 애니메이션 자체는 거의 생략하여 속도를 극대화). |

### B. 시각적 피드백 통제 (Visual Feedback Control)

목표: 불필요한 동적 움직임을 제거하고, 핵심 정보의 안정성을 확보한다.

*   **규칙 1: 정적 요소 우선:** 데이터가 로딩 중이거나 변경되지 않는 UI 요소(헤더, 내비게이션 바 등)는 애니메이션을 완전히 비활성화하여 시스템의 **정적인 안정성**을 강조한다.
*   **규칙 2: 상태 기반 움직임만 허용:** 동적 움직임은 오직 **상태 변화($Loading \to Success$)**에만 국한되어야 하며, 이는 $R_{Stability}$의 측정 지표(Latency)와 직접적으로 연결된다.
*   **규칙 3: 마이크로 인터랙션 제어:** 버튼 클릭이나 입력 시 발생하는 피드백은 즉각적이고 명확해야 한다. **느린 반응**을 유발하는 모든 시각적 지연(Visual Lag)을 제거하여, 사용자가 시스템의 응답 속도($C_{min}$)를 신뢰하게 만든다.
