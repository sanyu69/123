CEO의 지시사항을 확인했습니다. 기능 잠금(Feature Gating) 로직을 API 엔드포인트에 적용하여 시스템 안정성($R_{Stability}$)을 보장하는 것은 수익화 모델 성공의 필수 선행 조건입니다. 저는 이 목표를 달성하기 위한 상세 구현 계획과 기술적 실행 단계를 수립하겠습니다.

이 작업은 클라이언트 측 기능 잠금이 아닌, **서버 측 권한 검증 레이어**를 통해 구현되어야 하며, 이는 시스템 안정성 확보 원칙에 완벽히 부합합니다.

### 🛠️ 기능 잠금(Feature Gating) 상세 구현 계획 및 실행 단계

#### 1단계: 설계 및 데이터 매핑 (Design & Data Mapping)
목표는 IAP 티어와 API 엔드포인트 간의 명확한 매핑 규칙을 정의하는 것입니다.

1.  **IAP 티어 정의 확정:** `business` 에이전트가 제공한 가격 구조(Basic, Pro, VIP)를 기반으로 각 티어가 잠금(Gate)할 기능 목록($F_{basic}, F_{pro}, F_{vip}$)을 명확히 정의합니다.
2.  **$R_{Stability}$ 연관성 매핑:** 각 기능($F_i$)이 시스템 안정성에 미치는 영향을 분석하여, 특정 기능 잠금이 $R_{Stability}$ 지표에 어떻게 기여하는지 매핑 규칙을 확립합니다. (예: Pro 티어에서만 접근 가능한 고급 데이터는 $R_{Stability}$의 측정 정확도를 높임).
3.  **데이터 저장 구조 정의:** 사용자별 IAP 상태를 안전하게 관리할 데이터베이스 스키마(`user_subscriptions` 테이블 등)를 설계하고, 이 정보를 API 요청 시점에 신속하게 조회할 수 있도록 준비합니다.

#### 2단계: 서버 로직 구현 (Logic Implementation)
실제 기능 잠금 로직을 처리할 백엔드 미들웨어 또는 컨트롤러 레벨에 구현합니다.

1.  **권한 검증 서비스 구현:** 사용자의 구독 상태를 확인하고, 요청된 리소스($R$)가 해당 사용자에게 허용되는지($A$)를 검사하는 핵심 서비스 함수(`check_feature_access(userId, featureId)`)를 개발합니다.
2.  **API 미들웨어 적용 (Gate Implementation):** 모든 민감한 API 엔드포인트에 대해 요청이 컨트롤러에 도달하기 전에 이 권한 검증 로직을 삽입하는 미들웨어를 구현합니다.
    *   **흐름:** `Request In` $\rightarrow$ **Feature Gate Middleware** $\rightarrow$ `Business Logic` $\rightarrow$ `Response Out`
3.  **예외 처리 강화:** 기능이 잠겨있을 경우(Access Denied), 사용자에게 적절한 HTTP 상태 코드(예: 403 Forbidden)와 명확한 에러 메시지를 반환하도록 예외 처리 로직을 강화하여 $R_{Stability}$를 확보합니다.

#### 3단계: 테스트 및 안정성 검증 (Testing & Stability Verification)
구현된 로직이 예상대로 작동하며 시스템 안정성을 해치지 않는지 철저히 검증합니다.

1.  **단위 테스트 작성:** 권한 검증 서비스(`check_feature_access`)의 모든 시나리오(Basic, Pro, VIP 상태별 접근 시도)에 대해 단위 테스트를 작성하고 실행합니다.
2.  **통합 테스트 수행:** 실제 API 엔드포인트(예: `/api/v1/pro_feature`)에 대해 각 IAP 상태별로 접근을 시도하는 통합 테스트를 수행하여, 잠금이 정확하게 작동하는지 검증합니다.
3.  **성능 및 안정성 모니터링:** 기능 잠금 로직이 전체 API 응답 시간에 미치는 영향을 모니터링하고, 시스템 부하가 증가했을 때에도 $R_{Stability}$ 지표가 유지되는지 확인합니다.
