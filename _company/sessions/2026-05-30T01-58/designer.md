# IAP 기반 랜딩 페이지 CTA 및 오류 UX 흐름 (R_Stability 준수)

## 1. 목표
사용자가 선택한 IAP 티어에 따라 명확하게 제한된 기능만을 인지하고, 안정적인 경험을 제공하며, 잠금 해제(Upgrade)에 대한 동기를 부여하는 CTA 배치 및 오류 처리 흐름을 설계한다.

## 2. 핵심 원칙 (R_Stability 기반)
1. **투명성:** 모든 기능 제약 조건은 시각적으로 명확하게 표시되어야 한다.
2. **안정성 우선:** 잠금 상태(Gated State)는 혼란스럽지 않고, 시스템이 안정적으로 작동하고 있음을 강조해야 한다.
3. **신뢰 유지:** 오류 발생 시에도 사용자는 시스템의 신뢰를 잃지 않아야 하며, 해결책을 명확히 제시해야 한다.

## 3. IAP 티어별 CTA 배치 및 UX 흐름 상세

### A. Basic Tier (입문 단계)
- **목표:** 서비스 체험 유도 및 상위 티어 전환 동기 부여.
- **CTA 배치:** 메인 CTA는 '무료 체험 시작'에 집중하며, Pro/VIP 기능은 희미하게 표시(Ghosted)하여 잠재적 가치를 암시한다.
- **오류 UX (API 오류):** 서버 측 오류 발생 시, 시스템 안정성($R_{Stability}$)이 최우선임을 강조하는 메시지를 노출한다. (예: "잠시 후 안정화 중입니다. $R_{Stability}$ 점검 중...")

### B. Pro Tier (핵심 기능 단계)
- **목표:** 핵심 기능 접근의 명확한 제약 조건 제시 및 업그레이드 유도.
- **CTA 배치:** 'Pro 기능 잠금' 상태를 명시적으로 표시하며, 해당 기능을 활성화하기 위한 CTA($\rightarrow$ Upgrade)를 강조한다.
- **오류 UX (권한 오류):** 기능 잠금 실패 시, "이 기능은 Pro 티어에 속합니다. 업그레이드를 통해 잠금을 해제하세요."와 같이 구체적인 권한 메시지를 제공한다.

### C. VIP Tier (최상위 경험 단계)
- **목표:** 프리미엄 경험의 가치 극대화 및 커뮤니티 접근성 강조.
- **CTA 배치:** VIP 전용 기능에 대한 독점성을 강조하는 CTA를 배치하며, 안정적인 환경에서 프리미엄 경험을 누릴 수 있음을 암시한다.
- **오류 UX (결제/세션 오류):** 결제 또는 세션 관련 오류 시, 단순한 에러 메시지 대신 "VIP 시스템 안정성 확인 중입니다. 잠시 후 다시 시도해주세요."와 같이 고급 사용자에게 맞춘 안내를 제공한다.

## 4. 시각적 컴포넌트 명세 (Visual Component Specification)
- **State Mapping:** 각 티어에 따라 CTA 버튼의 색상, 그림자(Shadow), 텍스트 강조 수준을 $R_{Stability}$ 레벨에 따라 동적으로 변화시킨다.
    - Basic: Neutral Blue, Low Opacity Shadow.
    - Pro: Accent Color (Neon), Medium Opacity Shadow.
    - VIP: Premium Gold/Silver, Strong Contrast.
- **Error State Design:** 오류 메시지 영역은 항상 '안정성(Stability)'을 최우선으로 표시하는 톤앤매너를 유지한다. 에러 코드는 숨기고 사용자 친화적인 해결책 위주로 제시한다.
